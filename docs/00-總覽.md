# 00-總覽

## 這個專案解決什麼問題

`HK Tick Collector` 解決的是「把即時港股逐筆資料穩定落庫，且可長期維運」：

- 從 Futu OpenD 拉取資料（Push 優先、Poll 備援）。
- 用佇列（queue）與批次寫入處理吞吐尖峰。
- SQLite 採 WAL，兼顧寫入效率與查詢可用性。
- systemd + runbook + Telegram 告警，降低夜間值班成本。

## 給不同受眾的價值

### 交易研究員

- 可追溯 tick 原始序列（含 `seq` 與 `recv_ts_ms`）。
- 可快速做日內品質檢查與回放。

### 工程師

- 明確設定檔（`.env`）與測試基線（pytest + ruff）。
- `docker compose` 可快速體驗，不必先搞定 OpenD。

### SRE/運維

- systemd 模板可直接落地。
- `hk-tickctl` 一鍵查健康、查 DB、導出資料包。

## 你會在 repo 看到什麼

- `hk_tick_collector/`: 核心採集、持久化、通知邏輯。
- `deploy/`: systemd 模板、伺服器安裝與升級腳本。
- `scripts/`: 運維工具與資料導出工具。
- `docs/`: 本文件入口與 runbook。
