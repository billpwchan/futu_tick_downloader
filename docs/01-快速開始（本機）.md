# 01-快速開始（本機）

目標：在 5 分鐘內看到 DB row 成長與可查詢結果。

## 前置需求

- Docker / Docker Compose
- `make`（可選，但建議）

## 路徑 A：Mock 模式（推薦給第一次體驗）

```bash
git clone https://github.com/billpwchan/futu_tick_downloader.git
cd futu_tick_downloader
cp .env.example .env
docker compose --profile mock up -d --build mock-replay
```

驗證：

```bash
make db-stats
DATA_ROOT=./data/sqlite scripts/hk-tickctl db symbols --minutes 5
```

## 路徑 B：連接真實 OpenD

1. 修改 `.env`：

```dotenv
FUTU_HOST=<your-opend-host>
FUTU_PORT=11111
FUTU_SYMBOLS=HK.00700,HK.00981
DATA_ROOT=/data/sqlite/HK
```

2. 啟動 collector：

```bash
docker compose --profile opend up -d --build collector
```

3. 驗證：

```bash
make logs
make db-stats
```

## 常見問題

- 看不到資料：先確認 `FUTU_SYMBOLS` 不為空，且 OpenD 已授權。
- DB 路徑錯誤：確認 `DATA_ROOT` 與 compose volume 一致。
- 日誌很多 WARN：先跑 `scripts/hk-tickctl status` 看是否是連線問題。
