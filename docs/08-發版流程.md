# 08-發版流程

## 版本策略（SemVer）

- `MAJOR`：不相容變更
- `MINOR`：向後相容功能新增
- `PATCH`：向後相容修補

## Release Checklist

1. 確認 `main` 綠燈（CI 全部通過）
2. 更新 `CHANGELOG.md` 的 `Unreleased`
3. 本機執行：

```bash
make lint
make test
```

4. 建立 tag：

```bash
git tag v0.1.1
git push origin v0.1.1
```

5. 確認 GitHub Actions `Release` workflow 完成
6. 在 Release Notes 附上升級注意事項與回滾方式

## 發版後驗證

```bash
pip install --upgrade hk-tick-collector==<new-version>
hk-tick-collector --help || true
```

## 回滾

```bash
git checkout <last-known-good-tag>
sudo bash deploy/scripts/upgrade.sh
```
